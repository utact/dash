# 온보딩 프로세스 리뉴얼 제안서

**작성 일시**: 2026-01-03
**관련 도메인**: `onboarding`
**목표**: 사용자 경험(UX) 개선 및 저장소 연결 프로세스 마찰 제거

---

## 🧐 배경 및 현황 파악

**문제점**:
- 현재의 익스텐션 설치 및 저장소 연결 과정이 한 화면에서 이루어져 복잡함.
- 사용자가 익스텐션 설정을 완료하지 않고 넘어가는 경우, 저장소 정보를 제대로 읽어오지 못하는 문제가 발생.
- "설치 -> 설정 -> 갱신 -> 확인"의 흐름이 사용자에게 명확하지 않음.

**개선 방향**:
- 단계를 명확히 분리하여 사용자가 한 번에 하나의 태스크만 수행하도록 유도.
- 시스템이 사용자의 설정을 능독적으로 감지하고 확인받는 "Reverse Confirmation" 방식 도입.

---

## 🚀 새로운 온보딩 흐름 (7 Steps)

### Step 1. Solved.ac ID 입력
- **Goal**: 사용자 식별 및 데이터 수집의 기초 마련.
- **Action**: Solved.ac 핸들 입력 받기.
- **Validation**: 유효한 핸들인지 즉시 확인.

### Step 2. 정밀 분석 (Analysis)
- **Goal**: 사용자의 기량 분석.
- **Action**: 로딩 애니메이션 또는 진행바를 통해 분석 중임을 시각화.
- **System**: 백엔드에서 사용자 문제 풀이 데이터 분석.

### Step 3. 스터디 매칭 (Recommendation & Creation)
- **Goal**: 사용자 수준에 맞는 최적의 스터디 연결.
- **Action**:
    - AI 추천 스터디 카드 제시.
    - **[NEW]** 마음에 드는 스터디가 없을 경우를 대비한 **"새 스터디 만들기"** 버튼 추가.
- **User Choice**: 추천 스터디 가입 또는 신규 생성.

### Step 4. 익스텐션 설치 가이드 (Extension Guide)
- **Goal**: 자동 기록을 위한 필수 도구 설치 유도.
- **Critical Change**: 여기서 저장소 이름을 입력받지 않음. 오직 "설치"와 "익스텐션 내부 설정"에만 집중.
- **Guide Content**:
    1. "DashHub 익스텐션 설치하기" 버튼.
    2. "설치 후 익스텐션 아이콘을 눌러 GitHub 로그인과 저장소 생성을 완료해주세요" 라는 명확한 지시.
- **Waiting State**: 프론트엔드는 익스텐션이 설치되고 설정될 때까지 폴링하거나 "설정 완료" 버튼을 대기.

### Step 5. 저장소 감지 및 확인 (Repo Detection)
- **Goal**: 연결된 저장소 확인 및 확정.
- **Action**:
    - 시스템이 익스텐션 설정값(연동된 리포지토리)을 자동으로 읽어옴.
    - "감지된 저장소: `my-study-repo` 이(가) 맞나요?" 라고 묻는 확인 창 제시.
- **Fallback**: 감지 실패 시 수동 입력 또는 "다시 감지" 옵션 제공.

### Step 6. 사용 가이드 (Usage Guide)
- **Goal**: 서비스 작동 방식에 대한 멘탈 모델 형성.
- **Content**:
    - "이제 백준/SWEA에서 문제를 풀고 '제출'만 하세요."
    - "자동으로 `my-study-repo`에 커밋되고, 스터디 대시보드에 잔디가 심어집니다."
- **Visual**: 간단한 애니메이션이나 도식 활용 권장.

### Step 7. 대시보드 이동 (Complete)
- **Goal**: 온보딩 종료 및 메인 서비스 진입.
- **Action**: 환영 메시지와 함께 대시보드로 리다이렉트.

---

## 📝 기대 효과
- **마찰 감소**: 사용자는 복잡한 설정 값을 직접 입력할 필요 없이 클릭 위주로 진행.
- **오류 방지**: 시스템이 저장소를 감지하므로 오타나 잘못된 설정 방지.
- **학습 효과**: 사용 가이드를 통해 서비스 메커니즘을 명확히 이해하고 이탈률 감소.

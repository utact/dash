# BaekjoonHub Dash 개인정보 처리방침
마지막 수정일: 2025-11-26

---

### **소개**

- 본 확장 프로그램 "BaekjoonHub Dash"(이하 "확장")은 사용자가 알고리즘 문제 풀이 시 자신의 코드와 제출 이력을 GitHub 리포지토리에 자동으로 커밋/보관할 수 있도록 도와줍니다.
- 본 문서는 확장이 수집·저장·처리하는 정보의 종류, 사용 목적, 보관 방식, 사용자 권리 및 연락처 정보를 설명합니다.

---

### **수집하는 정보 및 처리 목적**

#### **인증정보**

- 수집 항목: GitHub OAuth 액세스 토큰(BaekjoonHub_token)
- 목적: GitHub API를 통한 리포지토리 생성, 리포 확인, 파일 생성/업데이트(커밋) 수행을 위해 필요합니다.
- 보관 위치: 사용자의 브라우저 내 chrome.storage.local(로컬 스토리지)에 저장됩니다.

#### **계정 식별정보 / 프로필**

- 수집 항목: GitHub 사용자명/ID(토큰을 통해 확인되는 정보), 플랫폼별 사용자명(필요 시 페이지에서 추출)
- 목적: 리포지토리 권한 확인, 커밋 작성자 확인, 사용자 인터페이스에 사용자 정보 표시

#### **웹사이트 콘텐츠 및 제출 관련 데이터**

- 수집 항목: 사용자가 방문한 지원 플랫폼(예: acmicpc.net, school.programmers.co.kr, swexpertacademy.com, level.goorm.io)에서 파싱한 문제 제목, 문제 메타데이터(출처 표기용), 사용자가 제출한 코드 스냅샷, 제출 결과(성공/실패), 제출 시각 등
- 목적: 사용자가 선택한 리포지토리에 자동으로 커밋을 생성하거나, 통계/아카이브(예: ZIP) 생성 및 UI 표시

#### **사용자 설정 및 확장 상태**

- 수집 항목: 확장 활성화 여부(bjhEnable), 연결된 리포지토리 식별자(BaekjoonHub_hook), 정렬 옵션(BaekjoonHub_OrgOption), 통계(stats) 등
- 목적: 사용자 설정 유지 및 정상 동작을 위한 로컬 구성 저장

#### **비수집(확인)**

- 본 확장은 건강정보, 금융·결제정보, 개인 메시지(이메일·채팅등), 실시간 위치 정보, 브라우저 전체 방문 기록(지원 호스트 도메인 외)은 수집하지 않습니다.

---

### **데이터 전송 및 제3자**

- GitHub API: 커밋/파일 업로드 등 작업은 GitHub API에 요청을 전송하며, 이 과정에서 커밋 대상 파일(사용자 코드, 문제 정보 등)과 인증 토큰이 GitHub으로 전송됩니다. 사용자가 연결한 리포지토리의 공개/비공개 설정에 따라 저장된 데이터의 접근 범위가 결정됩니다. (확장 기본 동작은 프라이빗 리포지토리 생성 옵션 제공)
- 기타 외부 API: 문제 보강용으로 solved.ac API 등을 조회할 수 있습니다. (읽기 전용 요청)
- 로컬 저장소(chrome.storage.local)에 저장된 데이터는 사용자의 브라우저에 남아 있으며, 확장 개발자 서버로 자동 전송되지 않습니다. (서버가 별도 존재하지 않는 한)

---

### **데이터 보관 기간**

- 로컬에 저장된 정보(토큰, 설정, 통계, 아카이브 등)는 사용자가 삭제하거나 확장을 제거할 때까지 보관됩니다.
- GitHub 리포지토리에 커밋된 파일은 해당 리포지토리의 보관 정책과 소유권에 따릅니다. (확장은 리포지토 내 파일을 생성/수정할 뿐, GitHub에 업로드된 데이터는 GitHub 서비스 규정에 따라 보관/관리됩니다.)

---

### **원격 코드 사용 여부**

- 이 확장은 런타임에 외부(원격) 자바스크립트 코드를 다운로드하여 실행하지 않습니다. 모든 필요한 스크립트(라이브러리 포함)는 확장 패키지에 번들(library 및 scripts 폴더)되어 있으며, manifest.json의 content_scripts도 로컬 파일을 참조합니다.
- 단, 확장은 GitHub, solved.ac 등 외부 API로부터 데이터를 조회(읽기)하고, 그 데이터를 바탕으로 로컬에서 처리하거나 GitHub에 업로드합니다.

---

### **보안 및 저장 방식**

- 인증 토큰 및 기타 민감 정보는 chrome.storage.local에 저장됩니다. 브라우저 프로필에 접근 가능한 사용자는 해당 데이터에 접근할 수 있으므로 공용/공유 장비에서는 주의가 필요합니다.
- 토큰과 같은 민감 정보는 최소 권한으로 사용되며, 확장은 토큰을 직접 외부 서버로 전송하지 않습니다(서버 미존재 시). GitHub API 호출을 위해 브라우저에서 직접 요청을 수행합니다.

---

### **사용자의 권리 및 제어 방법**

- 리포지토리 연결 해제: 확장 내 연결 해제 기능을 사용하거나, BaekjoonHub_hook 값을 null로 설정하면 확장이 더 이상 해당 리포지토리에 커밋을 생성하지 않습니다.
- 토큰 삭제/권한 철회: 확장에서 저장한 토큰을 삭제하려면 확장 설정(혹은 크롬 개발자 도구의 chrome.storage.local 항목)에서 BaekjoonHub_token을 제거하거나, GitHub의 OAuth 애플리케이션 권한 설정 페이지에서 액세스 토큰을 철회(revoke)할 수 있습니다.
- 로컬 데이터 삭제: 확장 설정에서 제공되는 초기화/언링크 기능 사용 또는 확장 제거(언인스톨) 시 chrome.storage.local에 저장된 모든 데이터가 제거됩니다. (브라우저 동작에 따름)
- GitHub에 업로드된 데이터 삭제: GitHub 리포지토리에 저장된 커밋/파일은 해당 리포지토리의 소유자 권한으로 삭제/수정해야 합니다.
